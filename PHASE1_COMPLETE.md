```typescript
async function fetchPosts() {
  setLoading(true)
  try {
    const params = new URLSearchParams({
      page: '1',
      limit: '10',
      ...(selectedCategory ? { category: selectedCategory } : {}),
    })
    
    console.log('Fetching posts with params:', params.toString())
    const res = await fetch(`/api/posts?${params}`)
    
    if (!res.ok) {
      throw new Error(`HTTP error! status: ${res.status}`)
    }
    
    const data = await res.json()
    console.log('Posts API response:', data)
    
    if (data.error) {
      throw new Error(data.error)
    }
    
    setPosts(data.posts || [])
  } catch (error) {
    console.error('Failed to fetch posts:', error)
    setPosts([]) // 优雅降级
  } finally {
    setLoading(false)
  }
}
```

## 📝 技术总结

### 经验教训

1. **API 正常但页面错误**
   - 原因：前端组件错误处理不足
   - 解决：增强 try-catch 和状态管理

2. **生产环境需要重新构建**
   - 原因：修改代码后未 rebuild
   - 解决：修改后必须 `npm run build`

3. **移动端测试很重要**
   - 发现：桌面端正常，移动端可能异常
   - 方法：DevTools + 真机测试

### 最佳实践

1. **错误处理**
   - 多层 try-catch
   - 详细日志
   - 优雅降级

2. **构建流程**
   - 修改代码 → 重新构建 → 重启服务 → 全面测试

3. **测试策略**
   - API 测试（curl）
   - 页面测试（HTTP 状态）
   - 移动端测试（DevTools）
