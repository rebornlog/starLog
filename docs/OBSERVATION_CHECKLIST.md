## 🚨 异常情况处理

### 情况 1：服务宕机

**现象：**
```
❌ 前端服务异常 (HTTP 503)
```

**处理：**
```bash
# 1. 查看自动恢复日志
tail -20 /tmp/smart-monitor.log

# 2. 手动检查服务
ps aux | grep "next"
ps aux | grep "uvicorn"

# 3. 手动重启（如需要）
cd /home/admin/.openclaw/workspace/starLog
npm start

# 4. 记录原因
echo "[$(date)] 服务宕机原因：..." >> /tmp/incident-log.md
```

---

### 情况 2：cron 未执行

**现象：**
```
定时任务未执行
```

**处理：**
```bash
# 1. 检查 cron 服务
systemctl status cron

# 2. 查看 cron 日志
grep CRON /var/log/syslog | tail -20

# 3. 重新安装 cron
./scripts/setup-cron.sh --install
```

---

### 情况 3：日志文件过大

**现象：**
```
日志文件 > 1GB
```

**处理：**
```bash
# 1. 清理旧日志
./scripts/analyze-logs.sh --clean

# 2. 配置日志轮转
# 编辑 /etc/logrotate.d/starlog
```
