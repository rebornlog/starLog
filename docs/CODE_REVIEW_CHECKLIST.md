## 🎯 审查原则

### 核心原则
1. **功能正确性** - 代码是否按预期工作
2. **性能优化** - 是否有性能问题
3. **安全性** - 是否有安全漏洞
4. **可维护性** - 代码是否易读易维护
5. **测试覆盖** - 是否有充分测试

---

## 📱 Next.js 前端审查

### 组件规范

- [ ] **组件职责单一** - 每个组件只做一件事
- [ ] **Props 类型定义** - 使用 TypeScript 或 PropTypes
- [ ] **避免内联样式** - 使用 Tailwind 或 CSS 模块
- [ ] **Key 属性** - 列表渲染必须有唯一 key
- [ ] **事件处理** - 使用 useCallback 避免重复创建
- [ ] **副作用清理** - useEffect 返回清理函数
- [ ] **避免直接 DOM 操作** - 使用 React refs

### 性能优化

- [ ] **图片优化** - 使用 next/image，添加 alt 文本
- [ ] **懒加载** - 非首屏内容使用动态导入
- [ ] **避免重复渲染** - 使用 React.memo、useMemo
- [ ] **代码分割** - 大组件拆分，按需加载
- [ ] **API 请求缓存** - 使用 SWR 或 React Query
- [ ] **减少 bundle 体积** - 检查依赖，移除未使用代码

### 状态管理

- [ ] **状态提升** - 共享状态提升到合适层级
- [ ] **避免全局状态滥用** - 优先使用局部状态
- [ ] **异步状态处理** - 处理 loading/error 状态
- [ ] **表单验证** - 前端验证 + 后端验证

### 可访问性 (a11y)

- [ ] **语义化 HTML** - 使用正确的标签
- [ ] **Alt 文本** - 所有图片有描述
- [ ] **键盘导航** - 支持 Tab 键导航
- [ ] **ARIA 标签** - 复杂组件添加 ARIA
- [ ] **颜色对比度** - 文本可读性

### 安全性

- [ ] **XSS 防护** - 不直接使用 dangerouslySetInnerHTML
- [ ] **CSRF 防护** - 表单提交携带 token
- [ ] **输入验证** - 用户输入前端验证
- [ ] **敏感信息** - 不将密钥暴露在前端

---

## 🐍 FastAPI 后端审查

### API 设计

- [ ] **RESTful 规范** - 使用正确的 HTTP 方法
- [ ] **路径命名** - 使用复数名词，小写 + 连字符
- [ ] **状态码** - 返回正确的 HTTP 状态码
- [ ] **错误处理** - 统一错误响应格式
- [ ] **分页** - 列表接口支持分页
- [ ] **过滤排序** - 支持常用过滤和排序

### 数据库操作

- [ ] **ORM 使用** - 使用 Prisma/SQLAlchemy
- [ ] **N+1 查询** - 使用 join/eager loading
- [ ] **索引优化** - 查询字段添加索引
- [ ] **事务处理** - 多步操作使用事务
- [ ] **连接池** - 配置合理的连接池大小
- [ ] **SQL 注入防护** - 使用参数化查询

### 性能优化

- [ ] **缓存策略** - 热点数据使用 Redis 缓存
- [ ] **异步处理** - 耗时任务异步执行
- [ ] **限流** - API 限流防止滥用
- [ ] **压缩** - 启用 Gzip/Brotli
- [ ] **数据库查询优化** - EXPLAIN 分析慢查询

### 安全性

- [ ] **认证授权** - JWT/OAuth2 验证
- [ ] **输入验证** - Pydantic 模型验证
- [ ] **密码加密** - 使用 bcrypt/argon2
- [ ] **CORS 配置** - 限制允许的域名
- [ ] **敏感信息** - 环境变量存储密钥
- [ ] **日志脱敏** - 不记录敏感信息

### 错误处理

- [ ] **统一异常处理** - 全局异常处理器
- [ ] **错误日志** - 记录详细错误信息
- [ ] **用户友好提示** - 错误消息清晰易懂
- [ ] **重试机制** - 临时失败自动重试

## 🔍 审查流程

### 自查清单（提交前）

1. [ ] 代码功能正常
2. [ ] 通过 Lint 检查
3. [ ] 通过测试
4. [ ] 更新文档
5. [ ] 无调试代码

### 同行审查（提交后）

1. [ ] 功能正确性审查
2. [ ] 代码质量审查
3. [ ] 性能影响评估
4. [ ] 安全性审查
5. [ ] 测试覆盖确认

### 合并前检查

1. [ ] 所有审查意见已解决
2. [ ] CI/CD 通过
3. [ ] 测试通过
4. [ ] 负责人批准
