## 🔴 P06: 列表页缓存读取失败（待修复）

### 问题描述

**现象：**
- Redis 缓存存在且数据完整
- API 仍返回静态数据（source: static）
- 基金代码为 000001, 000002...（静态数据）

**验证结果：**
```bash
# Redis 缓存正常
redis-cli GET "funds:list:all"
# ✅ 返回 19 只基金，净值数据完整

# API 返回异常
curl http://47.79.20.10:8081/api/funds/list?limit=5
# ❌ source=static, funds=000001-000005 (静态数据)
```

### 已排查项

1. ✅ **缓存键匹配** - 已统一为 `funds:list:{fund_type}`
2. ✅ **Redis 连接** - `REDIS_AVAILABLE = True`（强制启用）
3. ✅ **端口配置** - 前端已改为 8081 端口
4. ✅ **缓存数据** - Redis 中有完整数据
5. ❓ **get_cache 函数** - 待深入排查

### 日志分析

```
4|finance- | 获取基金 000001 详情失败：name 'get_fund_detail' is not defined
4|finance- | INFO: 47.79.20.10:48208 - "GET /api/funds/list?fund_type=all&limit=5 HTTP/1.1" 200 OK
```

**关键发现：**
- API 返回 200 OK
- 但返回的是静态数据（POPULAR_FUNDS）
- 缓存读取逻辑可能未执行

### 待排查方向

1. **get_cache 函数是否被调用**
   - 添加调试日志验证
   - 检查 REDIS_AVAILABLE 实际值

2. **缓存键格式是否匹配**
   - Redis: `funds:list:all`
   - API: `funds:list:{fund_type}` (fund_type=all)
   - 理论上匹配，但需验证

3. **异步函数执行问题**
   - get_fund_list 是 async 函数
   - get_cache 是同步函数
   - 可能存在执行顺序问题

**根因：** 缓存读取后又包装了一层 `{success, data}`

**修复：** 所有 Hooks 移到条件判断之前
**状态：** ✅ 已修复

**根因：** Next.js 开发模式长时间运行

**状态：** ✅ 已修复

### P05: 前端 API 端口错误
**根因：** 前端代码使用 8082 端口（应该是 8081）
