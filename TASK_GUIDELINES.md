## 🎯 核心准则

### 准则 1：及时同步 ⭐⭐⭐⭐⭐

**内容**: 每次完成任务之后，务必及时同步

**执行标准**:
1. ✅ 功能完成后立即 Git 提交
2. ✅ 提交后推送到 GitHub
3. ✅ 推送后生成进度报告
4. ✅ 报告包含完成内容、代码统计、测试结果

**同步流程**:
```
完成任务
   ↓
Git Add + Commit (描述清晰)
   ↓
Git Push (确保远程更新)
   ↓
生成进度报告 (包含统计)
   ↓
发送同步消息 (团队通知)
```

**违反后果**: 
- 信息不同步
- 团队协作效率降低
- 代码冲突风险增加

---

### 准则 2：代码质量 ⭐⭐⭐⭐⭐

**内容**: 所有代码必须经过测试和审查

**执行标准**:
1. ✅ 功能自测通过
2. ✅ 无 TypeScript 错误
3. ✅ 无 ESLint 警告
4. ✅ 代码格式统一
5. ✅ 注释清晰完整

**审查清单**:
- [ ] 功能是否按需求实现？
- [ ] 是否有边界情况处理？
- [ ] 是否有错误处理？
- [ ] 代码是否可读？
- [ ] 是否有性能问题？

---

### 准则 3：文档先行 ⭐⭐⭐⭐

**内容**: 功能开发前或同时完成文档

**执行标准**:
1. ✅ README 更新
2. ✅ 代码注释完整

### 准则 4：用户优先 ⭐⭐⭐⭐⭐

**执行标准**:
1. ✅ 加载状态友好
2. ✅ 错误提示清晰
3. ✅ 操作反馈及时
4. ✅ 移动端适配完善
5. ✅ 无障碍设计

- 错误可恢复

- 移动端可用
- 色盲友好

### 准则 5：性能意识 ⭐⭐⭐⭐

**内容**: 始终关注性能影响

**执行标准**:
1. ✅ 避免不必要的渲染
2. ✅ 合理使用缓存
3. ✅ 图片懒加载

5. ✅ 减少包体积

4. ✅ 敏感数据加密
5. ✅ 权限控制

**安全检查**:
- [ ] 用户输入是否验证？
- [ ] SQL 注入防护？
- [ ] XSS 防护？
- [ ] 敏感信息是否加密？
- [ ] 权限是否正确？

---

### 准则 7：可维护性 ⭐⭐⭐⭐

**内容**: 代码必须易于维护和扩展

**执行标准**:
1. ✅ 单一职责原则
2. ✅ DRY（Don't Repeat Yourself）
3. ✅ 清晰的命名
4. ✅ 合理的抽象
5. ✅ 完整的测试

**代码质量**:
- 函数长度 < 50 行
- 组件大小适中
- 依赖关系清晰
- 易于单元测试

---

### 准则 8：沟通透明 ⭐⭐⭐⭐⭐

**内容**: 保持信息透明和及时沟通

**执行标准**:
1. ✅ 进度及时更新
2. ✅ 问题及时上报
3. ✅ 决策记录在案
4. ✅ 变更通知团队

**沟通渠道**:
- 即时消息（飞书）
- Git 提交记录
- 进度报告
- 问题追踪（Issues）

## 📋 任务执行流程

### 标准流程

```
1. 接收任务
   ↓
2. 理解需求（确认目标）
   ↓
3. 制定计划（拆分步骤）
   ↓
4. 执行开发（按优先级）
   ↓
5. 自测验证（功能 + 边界）
   ↓
6. Git 提交（清晰描述）
   ↓
7. 推送 GitHub（确保同步）
   ↓
8. 生成报告（进度统计）
   ↓
9. 团队同步（发送消息）
   ↓
10. 等待反馈（准备迭代）
```
