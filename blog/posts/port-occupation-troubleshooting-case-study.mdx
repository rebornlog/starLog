## 故障现象

**时间：** 2026-03-25 08:16  
**发现人：** 水镜先生  
**影响：** 前端服务异常，PM2 显示重启 133 次

### 告警信息

```bash
$ pm2 list

│ id │ name                │ status    │ ↺    │
│ 0  │ starlog-frontend    │ waiting … │ 133  │
│ 1  │ finance-api         │ online    │ 2    │
│ 2  │ fund-api            │ online    │ 7    │
│ 3  │ api-proxy           │ online    │ 2    │
```

**关键信息：**
- 服务名：starlog-frontend
- 状态：waiting（等待中）
- 重启次数：133 次
- 其他服务正常

### 错误日志

```bash
$ pm2 logs starlog-frontend --lines 50

0|starlog- | Error: listen EADDRINUSE: address already in use :::3000
0|starlog- |     at Server.setupListenHandle [as _listen2] (node:net:1817:16)
0|starlog- |     at listenInCluster (node:net:1865:12)
0|starlog- |     at Server.listen (node:net:1953:7)
```

**错误分析：**
- 错误类型：`EADDRINUSE`
- 错误信息：端口 3000 已被占用
- 根本原因：PM2 无法绑定端口 3000

## 排查过程

### 第一步：确认端口占用

```bash
$ netstat -tlnp | grep 3000

tcp6       0      0 :::3000                 :::*                    LISTEN      349782/next-server
```

**发现：**
- 端口 3000 确实被占用
- 占用进程 PID：349782
- 进程名：next-server

### 第二步：检查进程来源

```bash
$ ps aux | grep 349782

admin  349782  1.5  2.5 2207964 197392 ?  Sl   04:00   4:05  next-server (v15.5.12)
```

**关键信息：**
- 启动用户：admin
- 启动时间：04:00（已运行 4 小时）
- 内存占用：197MB
- 进程类型：next-server（Next.js 服务）

### 第三步：判断进程归属

```bash
$ pm2 list | grep 349782

# 无输出
```

**结论：**
- 该进程**不是**PM2 管理的进程
- 是手动启动的"僵尸进程"
- 占用了端口 3000，导致 PM2 无法启动

### 第四步：分析重启循环

**PM2 重启逻辑：**
```
08:05:30  PM2 尝试启动 starlog-frontend
08:05:30  端口 3000 被占用，启动失败
08:05:35  PM2 自动重启（间隔 5 秒）
08:05:35  端口 3000 仍被占用，再次失败
08:05:40  PM2 再次重启
...
循环 133 次，直至今日
```

**为什么网站还能访问？**
- 僵尸进程（PID 349782）在正常运行
- 它占用了端口 3000，响应 HTTP 请求
- 但它不受 PM2 管理，无日志、无监控

## 根因分析

### 时间线还原

```
04:00  有人手动执行了 npm start 或 node server.js
       ↓
       启动了 Next.js 进程（PID 349782）
       ↓
       占用端口 3000
       ↓
08:05  PM2 尝试启动 starlog-frontend
       ↓
       端口 3000 已被占用
       ↓
       启动失败，PM2 自动重启
       ↓
       循环 133 次
```

### 为什么会手动启动？

**可能原因：**
1. 调试代码时手动启动
2. 测试功能时绕过 PM2
3. 忘记关闭手动启动的进程
4. 使用了`npm run dev`而不是`pm2 start`

### 为什么 PM2 不杀死旧进程？

**PM2 的工作机制：**
- PM2 只管理自己启动的进程
- 不会主动杀死非 PM2 管理的进程
- 避免误杀其他服务

**设计合理，但需要配合规范操作。**

## 解决方案

### 立即修复

```bash
# 1. 终止僵尸进程
$ kill -9 349782

# 2. 验证端口已释放
$ netstat -tlnp | grep 3000
# 无输出，说明端口已释放

# 3. 重启 PM2 服务
$ pm2 restart starlog-frontend

# 4. 验证服务正常
$ curl -I http://localhost:3000
HTTP/1.1 200 OK
```

**修复时间：** 2 分钟  
**服务恢复：** ✅

### 长期预防

#### 1. 创建端口清理脚本

```bash
#!/bin/bash
# scripts/cleanup-port.sh

PORT="${1:-3000}"

# 检查端口占用
PID=$(netstat -tlnp 2>/dev/null | grep ":$PORT " | awk '{print $7}' | cut -d'/' -f1)

if [ -n "$PID" ]; then
    # 检查是否为 PM2 进程
    PM2_ID=$(pm2 list | grep "$PID" | awk '{print $1}')
    
    if [ -n "$PM2_ID" ]; then
        pm2 stop $PM2_ID
    else
        kill -9 $PID
    fi
    
    echo "✅ 端口 $PORT 已清理"
fi
```

#### 2. 更新启动脚本

```bash
#!/bin/bash
# start-services.sh

# 启动前清理端口
cleanup_port 3000
cleanup_port 8081
cleanup_port 8082

# 检查僵尸进程
ZOMBIE_PIDS=$(ps aux | grep "next-server" | grep -v "grep" | grep -v "pm2" | awk '{print $2}')
for pid in $ZOMBIE_PIDS; do
    kill -9 $pid
done

# 启动 PM2 服务
pm2 start ecosystem.config.js
```

#### 3. 建立操作规范

**禁止行为：**
- ❌ 手动运行 `npm start`
- ❌ 手动运行 `npm run dev`
- ❌ 绕过 PM2 启动服务

**正确做法：**
- ✅ 使用 `pm2 start starlog-frontend`
- ✅ 使用 `pm2 restart all`
- ✅ 使用 `start-services.sh` 统一启动

## 工具脚本

### 端口清理脚本

```bash
#!/bin/bash
# scripts/cleanup-port.sh

PORT="${1:-3000}"

echo "🧹 清理端口 $PORT..."

PID=$(netstat -tlnp 2>/dev/null | grep ":$PORT " | awk '{print $7}' | cut -d'/' -f1)

if [ -n "$PID" ]; then
    PM2_ID=$(pm2 list | grep "$PID" | awk '{print $1}')
    
    if [ -n "$PM2_ID" ]; then
        echo "ℹ️  PM2 进程，先停止应用"
        pm2 stop $PM2_ID
    else
        echo "🔨 非 PM2 进程，直接终止"
        kill -9 $PID
    fi
    
    sleep 2
    
    if netstat -tlnp 2>/dev/null | grep -q ":$PORT "; then
        echo "❌ 端口仍然被占用"
        exit 1
    else
        echo "✅ 端口 $PORT 已释放"
    fi
else
    echo "✅ 端口 $PORT 未被占用"
fi
```

### 僵尸进程检测脚本

```bash
#!/bin/bash
# scripts/check-zombie.sh

echo "🧹 检查僵尸进程..."

ZOMBIES=$(ps aux | grep "next-server" | grep -v "grep" | grep -v "pm2" | awk '{print $2}')

if [ -n "$ZOMBIES" ]; then
    echo "⚠️  发现僵尸进程："
    for pid in $ZOMBIES; do
        echo "   PID: $pid"
        read -p "是否终止？(y/N): " -n 1 -r
        echo
        if [[ $REPLY =~ ^[Yy]$ ]]; then
            kill -9 $pid
            echo "   ✅ 已终止"
        fi
    done
else
    echo "✅ 未发现僵尸进程"
fi
```
